# Source notes

This skill combines broadly established product-design practice with independently paraphrased and operationalized lessons from selected public videos by Kole Jain. It contains no transcripts, copied passages, screenshots, Figma assets, or video assets.

## Method

- Use video titles, public descriptions, chapter topics, and independently synthesized lessons as research inputs.
- Convert useful ideas into task-oriented questions, decision rules, workflows, and verification criteria.
- Cross-check each idea against general usability, accessibility, and product-design practice before making it a durable rule.
- Treat visual trends, named aesthetics, and taste-based recommendations as contextual preferences.
- Do not imply that Kole Jain endorses this skill or every conclusion in it.

## Durable principles versus preferences

Treat these as **durable principles** because they remain useful across products and visual styles:

- Let user tasks and data shape the interface.
- Design connected flows and explicit states, not only idealized screens.
- Use progressive disclosure according to frequency, complexity, and risk.
- Make system status, semantic color, and interaction feedback consistent.
- Reuse a design system where it improves predictability and integrity.
- Choose navigation, overlays, pages, and components by responsibility and context.

Treat these as **contextual preferences** unless product evidence makes them functional requirements:

- A particular radius, shadow, blur, glow, gradient, or glass treatment
- A fashionable dashboard card composition
- A specific neutral tint or dark-mode aesthetic
- A named startup visual style
- Decorative motion or unusually sparse spacing
- Any component included mainly because admired products currently use it

When extending this skill, preserve this distinction and cite public sources without reproducing their expression or assets.
